# תכנית עבודה AGENT NOTEBOOKLM

## מטרה

להוכיח שאפשר לשלוט ב-NotebookLM מתוך OpenClaw דרך CLI/MCP ברמה שמצדיקה מעבר לתכנון סוכן מלא.

## היקף ה-POC

זה לא הסוכן הסופי. המטרה היא לבדוק את שרשרת הפעולות הבסיסית:

- התקנה
- התחברות
- קריאת מחברות
- יצירה ועריכה של מחברת בדיקה
- הוספת מקור
- שיחה עם מחברת

## שלבי עבודה

### 1. הקמת סביבת עבודה

- ליצור תיקייה ייעודית:
  `/root/.openclaw/workspace-Chloe-Speaker/DAVID/notebooklm-poc/`
- לבדוק זמינות של Node/Python/npm/pip/uv.
- לבדוק אם קיימת סביבת דפדפן או cookies רלוונטיים ל-Google login.

### 2. בחירת כלי ראשי

- להתחיל עם `notebooklm-mcp-cli`, כי הוא נראה הכי מתאים לחיבור CLI/MCP לסוכנים.
- אם הוא נכשל בהתקנה או auth, לבדוק fallback אחד בלבד: `notebooklm-py`.
- לתעד למה נבחר הכלי בפועל.

### 3. התקנה ובדיקת CLI

- להתקין את הכלי בסביבה מבודדת ככל האפשר.
- להריץ פקודת `--help` / `version`.
- לוודא שיש CLI פעיל, למשל `nlm --help` או המקבילה.

### 4. בדיקת התחברות ל-NotebookLM

- לזהות את שיטת ה-auth: cookies, browser session, token, או login flow.
- לא לשמור סיסמאות בקבצים.
- אם נדרש login ידני של דוד, לעצור ולדווח בדיוק מה צריך.
- אם אפשר להשתמש ב-session קיים, להמשיך.

### 5. בדיקות יכולת בסיסיות

לבצע רק על מחברת בדיקה:

- List notebooks: לוודא שאפשר לקרוא רשימת מחברות.
- Create notebook: ליצור מחברת בשם:
  `OpenClaw POC - NotebookLM`
- Rename notebook: לשנות שם ל:
  `OpenClaw POC - NotebookLM - Renamed`
- Add source: להוסיף מקור טקסט קצר שנוצר מקומית.
- Chat/query: לשאול שאלה פשוטה על המקור ולבדוק שהתשובה מבוססת עליו.
- Cleanup: למחוק את מחברת הבדיקה רק אם המחיקה ברורה ובטוחה. אחרת להשאיר ולדווח.

### 6. Discovery ליכולות מתקדמות

לא להפעיל פעולות כבדות עדיין. רק לבדוק אם הכלי חושף פקודות עבור:

- Deep research / web research
- Audio overview
- Video overview
- Mind map
- Report
- Quiz / flashcards
- Infographic
- Data table
- Sharing
- Download/export artifacts

3. אפשר לקרוא רשימת מחברות.
4. אפשר ליצור/לשנות מחברת או להוסיף מקור.
