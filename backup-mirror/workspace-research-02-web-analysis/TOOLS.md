# TOOLS.md - Local Notes

## Research Workspace Conventions

### Standard Output Root

- `research/` is the default root for research deliverables.
- `research/entities/` stores individual company or entity analysis files.
- `research/batches/` stores multi-company and task-level outputs.
- `research/sources/` stores source notes, captured references, and evidence logs when useful.
- `research/templates/` stores reusable markdown schemas and templates.

### Operating Rules

- Before every research task, reread [Website Analysis Agent Mission Statement.md](/root/.openclaw/workspace-research-02-web-analysis/Website%20Analysis%20Agent%20Mission%20Statement.md).
- Prefer browser-first and web-search-backed verification over recalled knowledge.
- For website analysis, save sparse but complete outputs rather than trimmed summaries.
- When David gives feedback after a task, create a lesson file under `lessions/` documenting the gap, the cause, and the fix.
